# 신발 기록 앱 (Shoes Log) — 설계 문서

- **작성일**: 2026-06-20
- **상태**: 승인됨 (구현 계획 작성 대기)

## 1. 개요

신발을 좋아하는 사용자를 위한 올인원 모바일 앱. 세 가지 기능을 하나로 묶는다:

1. **신발 컬렉션 관리** — 내가 가진 신발을 사진과 함께 등록·관리
2. **마일리지 추적** — 신발별 누적 거리를 기록해 교체 시기 추적
3. **사진 착화 일지** — 어떤 날 어떤 신발을 신었는지 사진과 메모로 기록

장기 목표: 본인 사용 + 앱스토어 개발자 등록/배포 경험.

## 2. 핵심 결정 사항

| 항목 | 결정 | 비고 |
|---|---|---|
| 플랫폼 | 네이티브 iOS/Android | |
| 기술 스택 | React Native + Expo + TypeScript | 한 코드로 양 플랫폼, 폰 즉시 테스트 |
| 데이터 저장 | 로컬 우선 (SQLite) | 나중에 클라우드 동기화 확장 대비 |
| 사진 저장 | 앱 전용 파일 저장소 + DB엔 경로만 | |
| 마일리지 입력 | 착화 일지 기록 시 거리 입력 → 신발별 누적 | |
| 교체 알림 | 신발별 목표 거리(선택), 임박/도달 시 앱 내 시각 표시 | 푸시는 나중 확장 |

## 3. 아키텍처

```
React Native (Expo)
  화면(UI)  →  상태/로직  →  데이터 계층
    • expo-router            (화면 이동)
    • expo-image-picker/camera (사진 촬영/선택)
    • expo-image-manipulator (리사이즈/압축)
    • expo-file-system       (사진 파일 저장)
    • expo-sqlite            (데이터베이스)
    • expo-notifications     (나중에 푸시용)
```

**핵심 원칙**: 데이터 계층(DB 접근)을 화면과 분리한다. 화면은 `getShoes()`, `addWearLog()` 같은 함수만 호출하고, 그 내부에서 지금은 SQLite를 다룬다. 나중에 클라우드 동기화를 붙일 때 화면 코드 변경을 최소화한다.

## 4. 데이터 모델

### shoes (신발)
| 필드 | 타입 | 설명 |
|---|---|---|
| id | INTEGER PK | 고유 번호 |
| name | TEXT (필수) | 이름/모델명 |
| brand | TEXT | 브랜드 |
| category | TEXT | 종류 (러닝화/운동화/구두/일상화 등) |
| photo_uri | TEXT | 대표 사진 경로 |
| purchase_date | TEXT | 구매일 |
| price | REAL | 가격 (선택) |
| target_distance | REAL | 교체 목표 거리(km, 선택, 비우면 마일리지 미추적) |
| retired | INTEGER (0/1) | 은퇴 여부 |
| created_at | TEXT | 등록 시각 |

### wear_logs (착화 일지)
| 필드 | 타입 | 설명 |
|---|---|---|
| id | INTEGER PK | 고유 번호 |
| shoe_id | INTEGER FK → shoes.id | 어떤 신발 |
| date | TEXT | 신은 날짜 |
| distance | REAL | 그날 거리(km, 선택) |
| memo | TEXT | 메모(코디/날씨/느낌 자유 입력) |
| created_at | TEXT | 기록 시각 |

### wear_log_photos (일지 사진, 1:N)
| 필드 | 타입 | 설명 |
|---|---|---|
| id | INTEGER PK | 고유 번호 |
| wear_log_id | INTEGER FK → wear_logs.id | 어떤 일지 |
| photo_uri | TEXT | 사진 파일 경로 |

**누적 거리는 저장하지 않고** `wear_logs.distance`의 합계로 계산한다(항상 정확, 데이터 꼬임 방지). 교체 판정 = `누적합 ≥ target_distance`(도달) 또는 임박 임계값 도달.

## 5. 화면 구성 (하단 탭 3개)

### 탭 1: 👟 내 신발 (컬렉션)
- 신발 카드/그리드 (사진 + 이름 + 누적거리 진행 막대)
- 교체 임박 🔴 배지, 목표 도달 시 "교체 권장" 표시
- 신발 상세: 정보, 누적 거리, 진행률 바, 해당 신발의 일지, 은퇴 버튼
- + 버튼 → 신발 등록 (사진, 이름·브랜드·종류·구매일·가격·목표거리)

### 탭 2: 📖 일지 (착화 기록)
- 날짜순 타임라인 (날짜, 신발, 거리, 사진 썸네일, 메모)
- + 버튼 → 일지 작성: 신발 선택, 날짜(기본 오늘), 거리(선택), 메모, 사진 여러 장
- 저장 시 해당 신발 누적 거리에 자동 반영
- 일지 상세: 사진 크게 보기 + 내용

### 탭 3: 📊 통계 (간단)
- 이번 달 총 거리, 가장 많이 신은 신발, 신발별 누적 거리 순위
- 첫 버전엔 가볍게, 나중에 확장/제거 용이

## 6. 핵심 사용 흐름

- **신발 등록**: + → 사진 → 정보 입력 → 저장 → 컬렉션 등장
- **착화 기록**: 일지 + → 신발 선택 → 거리 입력 → 사진 추가 → 저장 → 누적 자동 증가 → 임박 시 경고
- **교체**: 🔴 표시 확인 → "은퇴" 처리 (컬렉션 분리, 기록 보존)

## 7. 사진 처리

- 촬영/선택: expo-image-picker (카메라 또는 갤러리)
- 저장: 원본을 앱 전용 폴더에 복사, DB엔 경로만 (갤러리 원본 삭제돼도 유지)
- 용량: 저장 전 리사이즈/압축 (expo-image-manipulator)
- 권한: 최초 사용 시 카메라·사진 권한 요청, 거부 시 안내
- 정합성: 일지/신발 삭제 시 연결 사진 파일도 정리 (고아 파일 방지)

## 8. 에러 처리

- DB 작업 실패: 토스트 알림 + 롤백
- 사진 없음/깨짐: 플레이스홀더 이미지
- 입력 검증: 이름 필수, 거리·가격 숫자만, 음수 차단
- 권한 거부: "설정에서 권한 허용" 안내
